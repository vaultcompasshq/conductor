## Controls

- Positive control: staged README-only change against a contract that allowed README usage documentation and disallowed source changes.
- Negative control: staged source/package change with an explicit signal describing source drift.

## What We Learned

- The basic lifecycle works on real public repositories: `init`, `extract`,
  `freeze`, and `check` all completed across small packages, an application
  framework, and a monorepo.
- The negative control was reliable when `--signals` described what changed.
- Path-only drift is still weaker when the touched path is generic, such as
  `package.json`; v1 should improve diagnostics and reporting around that case.
- A repeatable public-repo validation harness should be promoted into
  `scripts/` before v1. The harness now lives at
  `scripts/validate-public-repos.mjs`.
